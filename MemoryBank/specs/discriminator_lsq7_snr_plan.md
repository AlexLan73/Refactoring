## 0. Контекст задачи

```
ФАР → ЛЧМ сигнал → Dechirp → Hanning → Zero-pad → FFT → Поиск пика → Дискриминатор
```

- Сигнал после FFT = **Hanning kernel**: `H(δ) = 0.5·sinc(δ) + 0.25·sinc(δ-1) + 0.25·sinc(δ+1)`
- Zero-pad ×2 → шаг между бинами = 0.5 bin
- Текущий лучший метод: **5QA** (SNR<25 дБ) или **5EA** (SNR>25 дБ)
- Цель: проверить, даёт ли МНК-7 дополнительный выигрыш

## 1. МНК-7 — аналитические формулы

### 1.1 Постановка задачи

7 равноотстоящих точек с шагом `h`:
```
x[i] = x_center + (i-3)·h,  i = 0..6
t[i] = i - 3              ∈ {-3,-2,-1,0,1,2,3}
```

Подбираем параболу: `y = a·t² + b·t + c` методом наименьших квадратов.

### 1.2 Замкнутые формулы (вывод через нормальные уравнения)

Для симметричной сетки t={-3,-2,-1,0,1,2,3}:
```
Σ1 = 7,  Σt = 0,  Σt² = 28,  Σt³ = 0,  Σt⁴ = 196
```

**Кривизна (a):**
```
84·a = Σtᵢ²·A[i] - 4·ΣA[i]
     = (9A₀ + 4A₁ + A₂ + 0·A₃ + A₄ + 4A₅ + 9A₆) - 4(A₀+A₁+A₂+A₃+A₄+A₅+A₆)
     = 5A₀ + 0·A₁ - 3A₂ - 4A₃ - 3A₄ + 0·A₅ + 5A₆

a = (5·A[0] - 3·A[2] - 4·A[3] - 3·A[4] + 5·A[6]) / 84
```

> ⚠️ **Замечание**: A[1] и A[5] имеют нулевые коэффициенты в `a`!
> Это значит, что оценка кривизны использует только A[0], A[2], A[3], A[4], A[6].
> A[1] и A[5] влияют только на наклон `b`.

**Наклон (b):**
```
b = (-3·A[0] - 2·A[1] - A[2] + A[4] + 2·A[5] + 3·A[6]) / 28
```

**Позиция пика:**
```
δ = -b / (2·a)              [в нормированных ед., δ ∈ [-0.5, +0.5] в идеале]
x_peak = x[3] + h · δ
```

### 1.3 7EA — Гауссова версия

Логарифмируем: `z[i] = log(A[i])`, затем те же формулы:
```
a_z = (5·z[0] - 3·z[2] - 4·z[3] - 3·z[4] + 5·z[6]) / 84
b_z = (-3·z[0] - 2·z[1] - z[2] + z[4] + 2·z[5] + 3·z[6]) / 28
δ   = -b_z / (2·a_z)
x_peak = x[3] + h · δ
```

Fallback: если `a_z >= -1e-30` или любой `A[i] ≤ 0` → использовать 7QA.

### 1.4 Сравнение с 5-точечными формулами

| Параметр | 5QA | 5EA | 7QA | 7EA |
|----------|-----|-----|-----|-----|
| Точек | 5 | 5 | 7 | 7 |
| Покрытие (zp×2) | ±2 бина | ±2 бина | ±3 бина | ±3 бина |
| Формула `a` | /14 | /14 | /84 | /84 |
| Формула `b` | /10 | /10 | /28 | /28 |
| A[1], A[5] в `a` | нет | нет | нет (!) | нет (!) |

### 2.2 Условия теста

```python
signal    = Hanning kernel:  H(δ) = 0.5·sinc(δ) + 0.25·sinc(δ-1) + 0.25·sinc(δ+1)
step      = 0.5 bin (zero-pad ×2)
shifts    = np.linspace(-0.225, 0.225, 21)   # 21 смещение в пределах ±0.5·step
n_trials  = 2000                              # Монте-Карло для каждого SNR
SNR_range = [5, 10, 15, 20, 25, 30, 35, 40] дБ
metric    = median(|δ_est - δ_true|)          # устойчивая к выбросам
```

### 2.3 Ожидаемые результаты (гипотезы)

**Гипотеза 1 — Стабильность при шуме:**
- 7pt > 5pt потому что больше точек → сглаживание шума лучше
- При SNR=20 дБ: ожидаем выигрыш ~1.2-1.5× над 5pt

**Гипотеза 2 — Bias на Hanning kernel:**
- Hanning kernel при step=0.5 хорошо аппроксимируется параболой вблизи центра
- Но на расстоянии ±1.5 бина уже заметнее отклонение → ожидаем бо́льший bias у 7pt vs 5pt

**Гипотеза 3 — Переключение:**
- На высоком SNR (>25 дБ): 7EA лучше 5EA из-за меньшего variance
- На низком SNR (<10 дБ): 7QA ≈ 5QA (bias доминирует, дополнительные точки мало помогают)

**Гипотеза 4 — Нулевые коэффициенты A[1], A[5]:**
- Соседи ±2h не входят в оценку кривизны напрямую → 7pt фактически использует "разреженный" паттерн {A[0], A[2], A[3], A[4], A[6]}
- Это означает, что шумовое усреднение для `a` — ровно 5 точек, как у 5pt!
- `b` использует все 7 — лучшая оценка наклона
- **Итог**: 7pt скорее всего лучше 5pt именно за счёт `b`, а не `a`

## 3. Оценка SNR — три подхода

### 3.1 Метод A: Спектральный шумовой пол (CFAR-стиль) ⭐ РЕКОМЕНДУЕМЫЙ

**Идея**: Бины FFT вдали от пика — чистый шум.

```python
# Имея полный спектр |X[k]|:
peak_bin = argmax(|X[k]|)
guard    = 5                          # защитная полоса [бин]
noise_window = 20                     # окно оценки шума [бин]

# Берём бины вне защитной полосы:
noise_bins = |X[peak_bin ± (guard..guard+noise_window)]|
sigma_noise = median(noise_bins)      # медиана устойчива к утечкам
SNR_est = 20 * log10(|X[peak_bin]| / sigma_noise)
```

**Плюсы:**
- ✅ Нет зависимости от формы сигнала (работает для любого окна)
- ✅ Нет насыщения (ограничение = размер FFT)
- ✅ Простейшая реализация
- ✅ Стандарт в радарной обработке (CFAR)

**Минусы:**
- ⚠️ Нужен доступ к широкому спектру (есть в нашем случае — полный FFT)
- ⚠️ Утечки от сильных мишеней рядом могут исказить оценку (решается guard band)

**Требования**: `peak_bin ± 25` бинов должны быть свободны → OK при шаге ±1000 бинов поиска.

---

### 3.2 Метод B: Невязки LSQ-7 (residuals)

**Идея**: После подгонки МНК-7 остатки содержат шум.

```python
# После МНК-7 получаем: a, b, c
# Восстанавливаем подогнанные значения:
t = np.array([-3,-2,-1,0,1,2,3])
y_fit = a*t**2 + b*t + c

# Оценка σ шума (n=7 точек, 3 параметра → 4 степени свободы):
residuals = A - y_fit
sigma_res = sqrt(sum(residuals**2) / 4)

# Оценка SNR:
SNR_est = 20 * log10(A[3] / sigma_res)   # A[3] = центральная точка = пик
```

**Плюсы:**
- ✅ Работает без внешних данных (только 7 точек)
- ✅ Просто реализовать одновременно с МНК

**Минусы:**
- ⚠️ **Насыщение**: Hanning kernel ≠ точная парабола → при высоком SNR остатки = систематическая ошибка модели, не шум
- ⚠️ Для 7pt: покрытие ±3 бина, дальше от центра парабола хуже приближает → насыщение ниже (~20 дБ)
- ⚠️ Для 5pt: насыщение ~15-16 дБ (по нашим предыдущим экспериментам)

**Вывод**: Полезен как быстрая диагностика, но не точный SNR.

---

### 3.3 Метод C: Отношение пика к соседям (простейший)

**Идея**: При низком SNR шум поднимает "дно" спектра рядом с пиком.

```python
# Для 5 точек (шаг h = 0.5 бина, zero-pad ×2):
# A = [A[-2], A[-1], A[0], A[1], A[2]]
# Теоретическое отношение для чистого Hanning kernel:
# A[0]:A[±1]:A[±2] ≈ 1 : 0.5·H(1) : 0.5·H(2)

# Просто используем отношение крайних к центру:
ratio = A[2] / (0.5 * (A[0] + A[4]))   # центр / среднее крайних

# Нормируем: для чистого сигнала ratio ≈ ratio_clean (вычислить заранее)
# SNR_rough = f(ratio)  — нелинейная функция, нужна калибровка
```

**Плюсы:**
- ✅ 1 формула, без итераций
- ✅ Можно вычислить одновременно с дискриминатором "бесплатно"

**Минусы:**
- ⚠️ Требует калибровки (ratio_clean зависит от шага)
- ⚠️ Грубая оценка, нелинейная
- ⚠️ Чувствителен к форме сигнала

## 6. Ожидаемые выводы по 7pt vs 5pt

**Оптимистичный сценарий** (7pt лучше):
- На SNR = 20-30 дБ: выигрыш 1.3-1.5× по сравнению с 5pt
- При очень малом step (zp×4): 7pt значительно лучше, т.к. 7 точек плотнее покрывают пик

**Пессимистичный сценарий** (7pt = 5pt):
- Коэффициент `a` не использует A[1], A[5] → эффективно 5 точек для кривизны
- Дополнительный bias из крайних точек A[0], A[6] может перекрыть выигрыш
- Итог: 7pt выигрывает только за счёт `b`, что незначительно

**Самый вероятный результат**:
- 7pt незначительно лучше 5pt при среднем SNR (15-25 дБ)
- На очень высоком SNR (>30 дБ): 7pt хуже из-за большего bias
- Практический вывод: **5EA + zp×2 остаётся рекомендуемым методом**,  
  7pt — академический интерес или для случаев zp×4
