Кодо! Ты супер-оркестратор проекта Refactoring. Твоя задача: создать и запустить цепочку sub-агентов для полного цикла рефакторинга C/C++ модуля.

---

## PROJECT_CONTEXT

- **Платформа**: Windows 11 (MSVC, VS 2026, C17) + Debian (GCC, C17/C11)
- **Языки**: C (рефакторинг на том же языке), C++ (обёртки, тесты, _new варианты)
- **Структура модуля**:
  ```
  [модуль]/
  ├── Old/               # Оригинальный код (нетронутый)
  ├── Doc/               # Документация (API.md, Full.md, Quick.md, Doxygen/, Review/)
  ├── include/           # Заголовочные файлы (.h)
  ├── src/               # Рефакторинг исходного кода (.c)
  ├── src_new/           # (опционально) C++ вариант с _new (если доказано лучше)
  ├── python/            # Python биндинги (ctypes / pybind11)
  ├── test_cpp/          # C++ тесты (*.hpp + all_test.hpp + README.md)
  ├── test_python/       # Python тесты (test_*.py)
  └── CMakeLists.txt     # Кросс-платформенная сборка
  ```
- **Правило рефакторинга**: НЕ менять интерфейс (.h)! Можно менять алгоритм если результат тот же.
- **Стиль**: Google C++ Style Guide, 2-пробельные отступы, Doxygen комментарии
- **Стиль написания**: ООП, SOLID, GRASP, GoF (для C++ обёрток)
---

## WORKFLOW

Запускай строго по шагам. После каждого — проверка. Если fail → resume agent и fix.

---

### Шаг 1 -- Analyzer агент

Анализ оригинального кода в `Old/`.

**Алгоритм:**
1. Прочитай все файлы в `Old/include/` и `Old/src/`
2. Разбери через sequential-thinking:
   - Что делает каждая функция (назначение, алгоритм)
   - Какие есть проблемы (стиль, возможные баги, оптимизации)
   - Какие граничные условия
3. Запиши анализ в `Doc/Review/yyyy-mm-dd_HH-min.md`

**Результат**: `"Анализ завершён: [список файлов], [количество функций], [список проблем]"`

---

### Шаг 2 -- Refactorer агент

Рефакторинг кода (на том же языке!).

**Алгоритм:**
1. Скопировать заголовки из `Old/include/` → `include/` (БЕЗ изменений интерфейса!)
2. Рефакторинг `Old/src/` → `src/`:
   - Стиль: 2-пробельные отступы, Doxygen комментарии
   - Оптимизация алгоритмов (если результат тот же)
   - Проверки индексов, граничных условий
3. Если C++ вариант доказано лучше → `src_new/` с суффиксом `_new` + описание ПОЧЕМУ
4. Создать `CMakeLists.txt`:
   ```cmake
   cmake_minimum_required(VERSION 3.15)
   project([module] LANGUAGES C CXX)
   set(CMAKE_C_STANDARD 17)
   set(CMAKE_C_STANDARD_REQUIRED OFF)  # fallback на C11
   ```

**Результат**: `"Рефакторинг: [список файлов], [что изменено], CMakeLists.txt готов"`

---

### Шаг 3 -- CodeReviewer агент (`/review`)

Ревью рефакторинга — сравнение Old/ vs новый код.

**Чеклист:**
- [ ] Интерфейс (.h) НЕ изменён
- [ ] Все функции сохранили семантику
- [ ] Нет новых багов (граничные условия)
- [ ] Стиль: 2-пробельные отступы, Doxygen
- [ ] CMakeLists.txt кросс-платформенный (MSVC + GCC)
- [ ] Код компилируется на Windows (MSVC) и Debian (GCC)

**Результат**: `"Ревью passed/failed: [список issues + fixes]"`

---

### Шаг 4 -- TestWriter_Cpp агент

C++ тесты для C-библиотеки.

**Структура тестов:**
```
test_cpp/
├── all_test.hpp            # Точка входа
├── test_[name]_basic.hpp   # Базовые тесты
├── test_[name]_edge.hpp    # Граничные случаи
├── Data/                   # Тестовые данные (если нужны)
└── README.md               # Описание тестов
```

**Паттерн:**
```cpp
extern "C" {
#include "../include/[header].h"
}
#include <cassert>
#include <cmath>
```

**Что тестировать:**
- Корректность: сравнение с ручным расчётом / NumPy
- Граничные условия: нулевые входы, максимальные значения
- Возвращаемые коды ошибок

**Результат**: `"Tests: X/Y passed"`

---

### Шаг 5 -- TestWriter_Python агент

Python тесты через ctypes.

**Алгоритм:**
1. Загрузить C-библиотеку через ctypes (.dll на Windows, .so на Debian)
2. Описать argtypes/restype для каждой функции
3. Сравнить результаты с NumPy/SciPy эталоном
4. Использовать PyCore: TestRunner + DataValidator

**Структура:**
```
test_python/
├── test_[module].py       # Основные тесты
└── test_[module]_plot.py  # Визуализация результатов (matplotlib)
```

**Результат**: `"Python tests: X/Y passed, plots saved to Results/Plots/"`

---

### Шаг 6 -- DocWriter агент (`/doc`)

Документация модуля.

**Создать:**
- `Doc/API.md` — краткий справочник API (сигнатуры, параметры, примеры)
- `Doc/Full.md` — полная документация (алгоритмы, формулы, диаграммы Mermaid)
- `Doc/Quick.md` — быстрый старт (1-2 примера)
- `Doc/Doxygen/Doxyfile` — конфигурация Doxygen
- Запустить `doxygen Doc/Doxygen/Doxyfile` → `Doc/Doxygen/html/`

**Обновить:**
- `MemoryBank/MASTER_INDEX.md` — добавить строку модуля
- `MemoryBank/tasks/COMPLETED.md` — записать завершение

**Результат**: `"Документация создана: [список файлов]"`

---

## ФИНАЛ (Meta-Orchestrator)

После всех шагов:
1. `cmake --build` — финальная проверка сборки (Windows + Debian)
2. Запустить C++ тесты и Python тесты
3. Сформировать git commit message:
   ```
   refactor([module]): рефакторинг [описание]

   - Код на C, интерфейс сохранён
   - C++ тесты: X/Y passed
   - Python тесты: X/Y passed (ctypes)
   - Документация: API.md, Full.md, Quick.md, Doxygen
   - Кросс-платформа: MSVC (C17) + GCC (C11/C17)
   ```

**Готово**: `"Полный цикл завершён: модуль [название] refactored + tested + documented"`
