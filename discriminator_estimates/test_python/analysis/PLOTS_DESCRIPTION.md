## Контекст: что такое дискриминатор координаты

**Задача.** Есть антенна (или РЛС/ГАС). Сигнал отражается от цели и принимается на
нескольких пространственных каналах (отсчётах). Каждый отсчёт — это амплитуда сигнала
в данной угловой или временно́й точке. По этим амплитудам нужно оценить, в каком
именно месте (угол, дальность, доплер) находится пик — то есть истинная координата цели.

**Форма сигнала** — диаграмма направленности антенны (ДН) или спектральный пик.
Типичная модель: `sinc(x) = sin(x)/x` или `Hanning kernel`.

**Дискриминатор** — алгоритм, который принимает на вход 2–9 отсчётов амплитуды
и возвращает оценку координаты пика `xe`. Истинная координата обозначается `x0`.
Ошибка оценки: `xe − x0`.

## Расшифровка сокращений

| Сокращение | Полное название | Пояснение |
|-----------|----------------|-----------|
| **CG** | Centre of Gravity — центр тяжести | `xe = Σ(Aᵢ·xᵢ) / Σ(Aᵢ)` — взвешенное среднее по амплитудам |
| **SD** | Sum-Difference — суммарно-разностный | `xe = xc + c·(A₂−A₁)/(A₂+A₁)` — смещение пропорционально разности каналов |
| **QA** | Quadratic Approximation — квадратичная аппроксимация | Парабола через 3 точки, вершина параболы = оценка пика |
| **EA** | Exponential Approximation — экспоненциальная аппроксимация | Гауссиан `exp(−x²/σ²)` через 3 точки; логарифмирование → парабола |
| **МНК** | Метод наименьших квадратов | Фит параболы/гауссиана к N точкам через систему нормальных уравнений |
| **5QA / 5EA** | МНК-парабола / МНК-Гауссиан по 5 точкам | Оптимальные коэффициенты для 5 равноотстоящих точек |
| **7QA / 7EA** | МНК-парабола / МНК-Гауссиан по 7 точкам | Аналогично для 7 точек |
| **9QA / 9EA** | МНК-парабола / МНК-Гауссиан по 9 точкам | Аналогично для 9 точек |
| **SNR / ОСШ** | Signal-to-Noise Ratio — отношение сигнал/шум | В дБ: `20·log₁₀(A/σ_noise)` |
| **bias** | Систематическая ошибка | Постоянная составляющая ошибки; не убирается усреднением |
| **СКО** | Среднеквадратическое отклонение | Стандартное отклонение ошибки по ансамблю испытаний |
| **x0** | Истинное положение пика | Задаётся при моделировании, обычно ∈ [−0.5, +0.5] шага сетки |
| **xe** | Оценка положения пика | Результат работы дискриминатора |
| **d** | Шаг сетки отсчётов | Угловое или временно́е расстояние между каналами |
| **NFFT** | Размер БПФ (FFT) | Количество точек быстрого преобразования Фурье |

## 📊 plots/2d/ — МНК-дискриминаторы по 5, 7, 9 точкам

Генерируется скриптом [09_lsq_compare.py](09_lsq_compare.py).
**Сигнал:** Hanning kernel — форма спектрального пика после Hanning окна:
`H(δ) = 0.5·sinc_n(δ) + 0.25·sinc_n(δ−1) + 0.25·sinc_n(δ+1)`

**Идея МНК-подхода:** вместо 3 точек (как в QA/EA) берём 5, 7 или 9 отсчётов
и решаем переопределённую систему методом наименьших квадратов. Больше точек →
лучше подавление шума, но нужна более широкая апертура.

- **NQA** — МНК-парабола: фитируем `a·x² + b·x + c` по N точкам, вершина = `xe`
- **NEA** — МНК-Гауссиан: логарифмируем амплитуды `log(Aᵢ)`, затем МНК-парабола

| Файл | Что показывает | Как читать |
|------|---------------|-----------|
| [lsq_bias_vs_x0.png](plots/2d/lsq_bias_vs_x0.png) | Знаковая ошибка `xe − x0` без шума | Систематический сдвиг при идеальном сигнале. EA-методы обычно точнее QA |
| [lsq_abs_error_vs_x0.png](plots/2d/lsq_abs_error_vs_x0.png) | `|xe − x0|` vs `x0`, лог. Y | Модуль ошибки. Показывает насколько хуже/лучше 7pt и 9pt относительно 5pt |
| [lsq_error_vs_snr.png](plots/2d/lsq_error_vs_snr.png) | Медиана `|xe − x0|` vs SNR (Монте-Карло, 500 испытаний) | Точность при шуме. Ожидается: 9EA > 7EA > 5EA при низком SNR |
| [lsq_gain_vs_snr.png](plots/2d/lsq_gain_vs_snr.png) | Выигрыш метода / 5EA vs SNR | Значение > 1 означает что метод точнее 5EA. Показывает при каком SNR выгодно использовать 7pt/9pt |

## 📊 plots/3d/ — Трёхмерные графики

Генерируется скриптом [05_plots_3d.py](05_plots_3d.py).
Показывают ошибку как функцию двух параметров: `x0` (позиция пика) и `d` (шаг сетки).

Суффикс `_{method}` — один из: `cg`, `sd`, `qa`, `ea`.

| Тип | Описание | Файлы |
|-----|---------|-------|
| **surface_error_{m}.png** | 3D-поверхность: ось X = x0, ось Y = d, ось Z = ошибка | [cg](plots/3d/surface_error_cg.png) · [sd](plots/3d/surface_error_sd.png) · [qa](plots/3d/surface_error_qa.png) · [ea](plots/3d/surface_error_ea.png) |
| **heatmap_error_{m}.png** | Тепловая карта (вид поверхности сверху), цвет = ошибка | [cg](plots/3d/heatmap_error_cg.png) · [sd](plots/3d/heatmap_error_sd.png) · [qa](plots/3d/heatmap_error_qa.png) · [ea](plots/3d/heatmap_error_ea.png) |
| **contour_error_{m}.png** | Контурные линии равной ошибки — удобно для выбора рабочей зоны | [cg](plots/3d/contour_error_cg.png) · [sd](plots/3d/contour_error_sd.png) · [qa](plots/3d/contour_error_qa.png) · [ea](plots/3d/contour_error_ea.png) |
| **surface_all_methods.png** | Все 4 метода на одном графике | [открыть](plots/3d/surface_all_methods.png) |

**Как читать:** тёмные области = малая ошибка (хорошо), светлые = большая.
На контурном графике выбирают зону, где ошибка не превышает заданного порога.
