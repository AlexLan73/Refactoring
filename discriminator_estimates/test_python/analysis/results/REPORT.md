## 1. Введение и постановка задачи

### Что такое дискриминатор?

В радиолокации **дискриминатор** — это алгоритм, который по нескольким отсчётам
диаграммы направленности (ДН) антенны восстанавливает **точное положение цели**
между дискретными отсчётами.

### 5 методов оценки

| Сокращение | Название | Суть | Точек | Исходник |
|------------|----------|------|-------|----------|
| **CG** | Центр тяжести | xe = Σ(Ai·xi) / Σ(Ai) | 2 лучших | discrcg.c |
| **SD** | Суммарно-разностный | xe = xc + c·(A2−A1)/(A2+A1) | 2 лучших | discrsd.c |
| **QA** | Квадратичная аппроксимация | Парабола → вершина (формула Ao) | все 3 | discrqa.c |
| **EA** | Экспоненциальная аппроксимация | Гауссиан exp(−a(x−x0)²) → x0 | все 3 | discrea.c |
| **AUTO** | Автоматический (EA + E2 + QA fallback) | Переключение по монотонности | все 3 | **discr_auto.c** |

> **AUTO** — новый метод! Автоматически выбирает лучшую стратегию:
> - Нормальные данные → EA (MAE = 0.005)
> - Монотонные данные → E2 Гауссова экстраполяция (MAE = 0.148)
> - EA failure → QA fallback

## 4. Как работает AUTO (discr3_auto)

### Статистика переключений (301 sweep-точка)

| Режим | Количество | % | Когда срабатывает |
|-------|-----------|---|-------------------|
| EA (mode=0) | 99 | 33% | Нормальные данные, есть перегиб |
| QA fallback (mode=1) | 0 | 0% | EA failure (не было в этом sweep) |
| E2 экстраполяция (mode=2) | 202 | 67% | Монотонные данные |
| E2 failure (mode=3) | 0 | 0% | Вырожденные данные (не было) |

### Использование в C-коде

```c
#include "discr_auto.h"

double xe;
int mode = discr3_auto(A1, A2, A3, x1, x2, x3, &xe);
/* mode: 0=EA, 1=QA fallback, 2=E2 экстраполяция, 3=E2 ошибка */
```

### Псевдокод алгоритма

```
если is_monotonic(A1, A2, A3):           ← 67% случаев!
    xe = extrap_gauss(A1, A2, A3, ...)   ← E2: MAE = 0.148
иначе:
    ret = discr3ea(A1, A2, A3, ...)      ← EA: MAE = 0.005
    если ret == FAILURE:
        xe = discr3qa(A1, A2, A3, ...)   ← QA: MAE = 0.006
```

## 5. Экстраполяция для монотонных случаев

| Метод | Суть | MAE | MaxErr |
|-------|------|-----|--------|
| E1 (парабола) | np.polyfit, clip | 0.324 | 1.230 |
| **E2 (Гауссиан)** | **log-парабола без проверки выпуклости** | **0.148** | **0.442** |
| E3 (градиент) | линейная регрессия + полшага | 0.500 | 1.000 |

> E2 = тот же алгоритм что EA, но БЕЗ проверки выпуклости и с расширенным clipping (2× вместо 0.5×).

## 8. Влияние шума (Monte Carlo, M=1000)

При x0 = 0.2:

| SNR (σ/Amax) | MAE CG | MAE QA | MAE EA | Лучший |
|:---:|--------|--------|--------|--------|
| 0.001 | 0.274 | 0.009 | **0.006** | EA |
| 0.01 | 0.274 | 0.023 | **0.023** | EA ≈ QA |
| 0.05 | 0.294 | 0.193 | **0.150** | EA |
| 0.1 | 0.354 | 0.938 | **0.324** | EA |
| 0.2 | **0.371** | 1.436 | 0.520 | CG! |

> При сильном шуме QA "взрывается" (формула Ao нестабильна). AUTO использует EA, не QA.

## 11. Рекомендации

| Ситуация | Метод | Причина |
|----------|-------|---------|
| **Универсальный выбор** | **AUTO (discr3_auto)** | **Сам переключается EA↔E2** |
| Штатный режим, мало шума | EA | MAE = 0.005 |
| Быстрая оценка | CG | 1 деление |
| FFT-спектр | EXP + Kaiser(14) | MAE = 471 Гц |

**Главная рекомендация**: использовать `discr3_auto()` — один вызов вместо
ручной проверки монотонности и выбора метода.

## 12. Выводы

1. **AUTO = лучший универсальный метод**: EA в normal зоне + E2 в extreme зоне
2. **EA — безусловный лидер** по точности (MAE = 0.005, 0.5% шага)
3. **67% случаев — монотонные**, без AUTO они обрабатываются плохо
4. **QA нестабилен** при шуме (формула Ao "взрывается") и при большом шаге
5. **CG/SD — грубые** методы (~20% ошибка), но робастные
6. **Kaiser(β=14)** улучшает FFT-оценку EXP в 6 раз
