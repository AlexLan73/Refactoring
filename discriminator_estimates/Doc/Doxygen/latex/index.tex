\hypertarget{index_autotoc_md48}{}\doxysection{\texorpdfstring{Что это и зачем}{Что это и зачем}}\label{index_autotoc_md48}
Эта библиотека отвечает на один очень практичный вопрос\+:

\begin{quote}
{\bfseries{«У меня есть несколько точек измерения сигнала. Где находится истинный максимум — между точками?»}} \end{quote}
Под «точками» понимаются\+:
\begin{DoxyItemize}
\item отсчёты диаграммы направленности (ДН) антенны для оценки {\bfseries{угловых координат цели}} в РЛС;
\item бины спектра FFT для оценки {\bfseries{точной частоты}} одиночного гармонического сигнала в зашумлённой записи.
\end{DoxyItemize}

В обоих случаях математика одинаковая, и здесь собраны {\bfseries{6 проверенных методов}} — от классических 2-\/точечных до новых 5-\/точечных МНК-\/аппроксимаций.\hypertarget{index_autotoc_md49}{}\doxysubsection{\texorpdfstring{Аналогия из жизни}{Аналогия из жизни}}\label{index_autotoc_md49}
\begin{quote}
Представь, что ты стоишь на стадионе в шумной толпе и пытаешься услышать, на какой ноте поёт один конкретный певец. Толпа — это шум. Голос певца — наш полезный сигнал. У тебя есть три-\/пять микрофонов, расставленных по частотной шкале — каждый «слышит» немного по-\/разному. Задача\+: по этим 3–5 числам угадать, где {\bfseries{точно}} находится голос певца.

То, чем занимается этот модуль — это и есть «найти певца в толпе». \end{quote}
Подробное объяснение для чайника + полная теория есть в файле {\ttfamily Doc/\+Frequency\+\_\+\+Recommendations\+\_\+2026-\/04-\/06.\+md} (на этом же документе и основан большой раздел \doxylink{recommendations_page}{Итоговые рекомендации}).
