Полный pipeline обработки сигнала: от сырых отсчётов до точной частоты

Что это и зачем

Эта библиотека отвечает на один очень практичный вопрос:

‍«У меня есть несколько точек измерения сигнала. Где находится истинный максимум — между точками?»

Под «точками» понимаются:

  • отсчёты диаграммы направленности (ДН) антенны для оценки угловых координат цели в РЛС;
  • бины спектра FFT для оценки точной частоты одиночного гармонического сигнала в зашумлённой записи.

В обоих случаях математика одинаковая, и здесь собраны 6 проверенных методов — от классических 2-точечных до новых 5-точечных МНК-аппроксимаций.

Аналогия из жизни

‍Представь, что ты стоишь на стадионе в шумной толпе и пытаешься услышать, на какой ноте поёт один конкретный певец. Толпа — это шум. Голос певца — наш полезный сигнал. У тебя есть три-пять микрофонов, расставленных по частотной шкале — каждый «слышит» немного по-разному. Задача: по этим 3–5 числам угадать, где точно находится голос певца.

То, чем занимается этот модуль — это и есть «найти певца в толпе».

Подробное объяснение для чайника + полная теория есть в файле Doc/Frequency_Recommendations_2026-04-06.md (на этом же документе и основан большой раздел Итоговые рекомендации).

Главная рекомендация (на одну страницу)

Дерево решений: какой метод выбрать

‍Если кратко: для FFT-задачи (поиск частоты) используй **discr5ea с fallback на discr5qa** и обязательным zero-padding ×2 с окном Hanning.

**Это даёт ошибку ~0.6 Гц при SNR = 20 дБ для типичного радара (fs = 100 МГц, N = 4M) — в 20 раз лучше, чем без интерполяции, и в 2 раза лучше классического 3-точечного discr3ea.**

Подробное обоснование — на странице Итоговые рекомендации.

Демонстрация работы

Анимация: автоматический выбор метода в разных сценариях (sweep + шум + step)

Что видно на анимации:

  • зелёный пунктир — истинное положение пика (то, что надо найти);
  • цветные вертикальные линии — оценки разных методов;
  • точки сверху — отсчёты, которые «видит» алгоритм;
  • хорошие методы сидят рядом с зелёной линией, плохие — далеко.

Больше анимаций — на странице Визуальный разбор методов.

Шесть методов одним взглядом

Метод Файл Точек Идея Краткая формула Когда хорош
CG discrcg.h 2…N Центр тяжести \( x_e = \frac{\sum A_i x_i}{\sum A_i} \) Грубая оценка
SD discrsd.h 2 Сумма-разность (моноимпульсный) \( x_e = x_c + c \cdot \frac{A_2 - A_1}{A_2 + A_1} \) 2 канала, моноимпульс
QA discrqa.h 3 Парабола по 3 точкам \( x_e = -\frac{b}{2a} \) Параболоподобный пик
EA discrea.h 3 Парабола в log-масштабе (Гауссиан) \( x_e = \alpha / (2\beta) \) Гауссовый пик
5QA ★ discr5ea.h 5 МНК-парабола по 5 точкам \( a = \dfrac{2y_1 - y_2 - 2y_3 - y_4 + 2y_5}{14} \) Шум, Hanning kernel
5EA ★★ discr5ea.h 5 МНК-Гауссиан по 5 точкам то же, но \( z_i = \ln y_i \) Универсал, FFT

★ — добавлено при рефакторинге 2026, ★★ — рекомендуемый по умолчанию для FFT.

Быстрый старт

C: суб-бинная оценка частоты по 5 точкам спектра (рекомендация)

#include "discr5ea.h"
#include <complex.h>
// X[] — комплексный спектр после FFT (длина N_padded)
// k_max — индекс максимума |X(k)| (грубая оценка)
// fs — частота дискретизации [Гц]
// N_padded — длина FFT с zero-padding ×2
double y[5], xs[5];
for (int i = 0; i < 5; ++i) {
y[i] = cabs(X[k_max - 2 + i]); // |X(k-2)| .. |X(k+2)|
xs[i] = (double)(i - 2); // {-2, -1, 0, 1, 2}
}
double delta;
int ok = discr5ea(y, xs, &delta); // МНК-Гауссиан по 5 точкам
if (ok != 0) {
// fallback: какая-то |X(k)| ≤ 0 → 5QA по амплитудам
discr5qa(y, xs, &delta);
}
double f_hz = (k_max + delta) * fs / (double)N_padded;
