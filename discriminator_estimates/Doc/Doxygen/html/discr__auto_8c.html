Реализует стратегию автопереключения на основе результатов аналитического исследования (analysis/REPORT.md):

Нормальный случай (есть перегиб): EA → QA fallback EA даёт MAE = 0.005 (0.5% шага сетки)

Монотонный случай (пик за сеткой): E2 (Гауссова экстраполяция) E2 даёт MAE = 0.148, что в 2-3 раза лучше прямого EA

E2 — это тот же алгоритм что и EA (логарифмирование + парабола), но БЕЗ проверки на выпуклость и с расширенным clipping (2× вместо 0.5×). Это позволяет вершине параболы оказаться далеко за сеткой — что и нужно.

Гауссова экстраполяция (E2) для монотонных случаев.

Алгоритм — как discr3ea, но без проверки выпуклости и с расширенным диапазоном допустимого вылета (2× вместо 0.5× от размера сетки).

Логарифмирует амплитуды, аппроксимирует параболой в log-пространстве, находит вершину. Для монотонных данных вершина за пределами сетки — это ожидаемое и полезное поведение.

Из исследования: E2 MAE = 0.148 для монотонных случаев, что в 2.2 раза лучше E1 (парабола, MAE = 0.324) и в 3.4 раза лучше E3 (градиент, MAE = 0.500).

A2амплитуда второй точки (должна быть > 0)

136 *
137 * Парабола z(x) через 3 точки (f1,z1), (f2,z2), (f3,z3):
138 * a_coef = z1*(f2²-f3²) + z2*(f3²-f1²) + z3*(f1²-f2²)
139 * b_coef = z1*(f2-f3) + z2*(f3-f1) + z3*(f1-f2)

141 * Вершина: xe = 0.5 * a_coef / b_coef

161 /* Вершина параболы в log-пространстве = оценка положения пика */
162 *xe = 0.5 * a_coef / b_coef;
163

166 *
167 * Расширенный диапазон: ±2 × размер_сетки (вместо ±0.5× в discr3ea).

169 *
170 * Почему 2×? Из исследования: при вылете > 2× шага ошибка
171 * экстраполяции уже не уменьшается, а растёт.
172 */

175 if (*xe > f3 + span) {
176 *xe = f3 + span;
177 return EXIT_FAILURE;
178 }

182 }

Проверка монотонности трёх отсчётов.

Монотонные данные = нет локального максимума среди 3 точек. Это означает, что пик за пределами сетки.
