## Что это и зачем

Эта библиотека отвечает на один очень практичный вопрос:

> **«У меня есть несколько точек измерения сигнала. Где находится истинный
> максимум — между точками?»**

Под «точками» понимаются:
- отсчёты диаграммы направленности (ДН) антенны для оценки **угловых
  координат цели** в РЛС;
- бины спектра FFT для оценки **точной частоты** одиночного гармонического
  сигнала в зашумлённой записи.

В обоих случаях математика одинаковая, и здесь собраны **6 проверенных
методов** — от классических 2-точечных до новых 5-точечных МНК-аппроксимаций.

### Аналогия из жизни

> Представь, что ты стоишь на стадионе в шумной толпе и пытаешься услышать,
> на какой ноте поёт один конкретный певец. Толпа — это шум. Голос певца —
> наш полезный сигнал. У тебя есть три-пять микрофонов, расставленных по
> частотной шкале — каждый «слышит» немного по-разному. Задача:
> по этим 3–5 числам угадать, где **точно** находится голос певца.
>
> То, чем занимается этот модуль — это и есть «найти певца в толпе».

Подробное объяснение для чайника + полная теория есть в файле
`Doc/Frequency_Recommendations_2026-04-06.md` (на этом же документе и
основан большой раздел @ref recommendations_page).

## Главная рекомендация (на одну страницу)

@image html 06_decision_tree.png "Дерево решений: какой метод выбрать"

> **Если кратко:** для FFT-задачи (поиск частоты) используй
> **`discr5ea` с fallback на `discr5qa`** и обязательным `zero-padding ×2`
> с окном Hanning.
>
> **Это даёт ошибку ~0.6 Гц при SNR = 20 дБ для типичного радара
> (`fs = 100 МГц`, `N = 4M`) — в 20 раз лучше, чем без интерполяции,
> и в 2 раза лучше классического 3-точечного `discr3ea`.**

Подробное обоснование — на странице @ref recommendations_page.

## Шесть методов одним взглядом

| Метод | Файл | Точек | Идея | Краткая формула | Когда хорош |
|-------|------|-------|------|-----------------|------------|
| **CG**  | `discrcg.h`  | 2…N | Центр тяжести | \f$ x_e = \frac{\sum A_i x_i}{\sum A_i} \f$ | Грубая оценка |
| **SD**  | `discrsd.h`  | 2 | Сумма-разность (моноимпульсный) | \f$ x_e = x_c + c \cdot \frac{A_2 - A_1}{A_2 + A_1} \f$ | 2 канала, моноимпульс |
| **QA**  | `discrqa.h`  | 3 | Парабола по 3 точкам | \f$ x_e = -\frac{b}{2a} \f$ | Параболоподобный пик |
| **EA**  | `discrea.h`  | 3 | Парабола в `log`-масштабе (Гауссиан) | \f$ x_e = \alpha / (2\beta) \f$ | Гауссовый пик |
| **5QA** ★ | `discr5ea.h` | 5 | МНК-парабола по 5 точкам | \f$ a = \dfrac{2y_1 - y_2 - 2y_3 - y_4 + 2y_5}{14} \f$ | Шум, Hanning kernel |
| **5EA** ★★ | `discr5ea.h` | 5 | МНК-Гауссиан по 5 точкам | то же, но \f$ z_i = \ln y_i \f$ | **Универсал, FFT** |

★ — добавлено при рефакторинге 2026, ★★ — **рекомендуемый по умолчанию** для FFT.

## Быстрый старт

### C: суб-бинная оценка частоты по 5 точкам спектра (рекомендация)

```c
#include "discr5ea.h"
#include <complex.h>

// X[]    — комплексный спектр после FFT (длина N_padded)
// k_max  — индекс максимума |X(k)| (грубая оценка)
// fs     — частота дискретизации [Гц]
// N_padded — длина FFT с zero-padding ×2

double y[5], xs[5];
for (int i = 0; i < 5; ++i) {
  y[i]  = cabs(X[k_max - 2 + i]);   // |X(k-2)| .. |X(k+2)|
  xs[i] = (double)(i - 2);          // {-2, -1, 0, 1, 2}
}

double delta;
int ok = discr5ea(y, xs, &delta);   // МНК-Гауссиан по 5 точкам
if (ok != 0) {
  // fallback: какая-то |X(k)| ≤ 0 → 5QA по амплитудам
  discr5qa(y, xs, &delta);
}

double f_hz = (k_max + delta) * fs / (double)N_padded;
```

### C: классический 3-точечный QA на угловой ДН

```c
#include "discrqa.h"

double A1 = 0.75, A2 = 2.75, A3 = 2.75;
double x1 = -1.0, x2 = 0.0,  x3 = 1.0;
double xe = discr3qa(A1, A2, A3, x1, x2, x3);
// xe ≈ 0.5 — оценка положения пика между точками
```

### C: автоматический выбор с защитой от монотонной зоны

```c
#include "discr_auto.h"

double xe;
int how = discr3_auto(A1, A2, A3, x1, x2, x3, &xe);
// how == 0 → EA использован   (нормальный случай)
// how == 1 → QA fallback      (EA не сошёлся)
// how == 2 → E2 экстраполяция (монотонные данные)
// how == 3 → E2 не сошёлся    (xe = x2)
```
