==========================
Discriminator Estimates
==========================

.. rubric:: Дискриминаторные оценки координат — поиск пика в зашумлённом сигнале

Полная документация модуля для определения частоты/координаты пика
в FFT-спектре или ДН антенны с **суб-бинной точностью**.

:Модуль: ``discriminator_estimates``

О чём этот модуль
=================

Библиотека отвечает на один очень практичный вопрос:

    **«У меня есть несколько точек измерения сигнала. Где находится
    истинный максимум — между точками?»**

Под «точками» понимаются:

- отсчёты **диаграммы направленности** (ДН) антенны для оценки угловых
  координат цели в РЛС;
- **бины спектра FFT** для оценки точной частоты одиночного
  гармонического сигнала в зашумлённой записи.

В обоих случаях математика одинаковая, и здесь собраны **6 проверенных
методов** — от классических 2-точечных до новых 5-точечных МНК-аппроксимаций.

.. admonition:: Аналогия из жизни
   :class: note

   Представь, что ты стоишь на стадионе в шумной толпе и пытаешься
   услышать, на какой ноте поёт один конкретный певец. Толпа — это шум.
   Голос певца — наш полезный сигнал. У тебя есть 3–5 микрофонов,
   расставленных по частотной шкале. Задача: по этим 3–5 числам угадать,
   где **точно** находится голос певца.

Главная рекомендация
====================

.. tip::

   **Используй** ``discr5ea`` **(МНК-Гауссиан по 5 точкам) с fallback
   на** ``discr5qa`` **и обязательным zero-padding ×2 + окном Hanning.**

   Это даёт точность ~0.6 Гц при SNR = 20 дБ — **в 20 раз лучше**, чем
   без интерполяции, и в 2 раза лучше классической 3-точечной ``discr3ea``.

Быстрый старт
=============

Суб-бинная оценка частоты по 5 точкам спектра (рекомендация):

.. code-block:: c

   #include "discr5ea.h"
   #include <complex.h>
   #include <stdlib.h>

   // X[]      — комплексный спектр после FFT (длина N_padded)
   // k_max    — индекс максимума |X(k)| (грубая оценка)
   // fs       — частота дискретизации [Гц]
   // N_padded — длина FFT с zero-padding ×2

   double y[5], xs[5];
   for (int i = 0; i < 5; ++i) {
     y[i]  = cabs(X[k_max - 2 + i]);   // |X(k-2)| .. |X(k+2)|
     xs[i] = (double)(i - 2);          // {-2, -1, 0, 1, 2}
   }

   double delta;   // суб-бинное смещение в единицах padded-шага
   int ok = discr5ea(y, xs, &delta);   // МНК-Гауссиан по 5 точкам
   if (ok != EXIT_SUCCESS) {
     // fallback: какая-то амплитуда ≤ 0 → 5QA по амплитудам
     discr5qa(y, xs, &delta);
   }

   double f_hz = (k_max + delta) * fs / (double)N_padded;

Классический 3-точечный QA на угловой ДН:

.. code-block:: c

   #include "discrqa.h"

   double A1 = 0.75, A2 = 2.75, A3 = 2.75;
   double x1 = -1.0, x2 = 0.0,  x3 = 1.0;
   double xe = discr3qa(A1, A2, A3, x1, x2, x3);
   // xe ≈ 0.5 — оценка положения пика между точками

Шесть методов одним взглядом
============================

.. list-table::
   :header-rows: 1
   :widths: 10 15 8 20 25 22

   * - Метод
     - Файл
     - Точек
     - Идея
     - Краткая формула
     - Когда хорош
   * - **CG**
     - :file:`discrcg.h`
     - 2…N
     - Центр тяжести
     - :math:`x_e = \dfrac{\sum A_i x_i}{\sum A_i}`
     - Грубая оценка
   * - **SD**
     - :file:`discrsd.h`
     - 2
     - Сумма-разность
     - :math:`x_e = x_c + c\,\dfrac{A_2 - A_1}{A_2 + A_1}`
     - Моноимпульсный канал
   * - **QA**
     - :file:`discrqa.h`
     - 3
     - Парабола по 3 точкам
     - :math:`x_e = -\dfrac{b}{2a}`
     - Параболоподобный пик
   * - **EA**
     - :file:`discrea.h`
     - 3
     - Парабола в log-масштабе (Гаусс)
     - :math:`x_e = \alpha / (2\beta)`
     - Гауссовый пик
   * - **5QA** ★
     - :file:`discr5ea.h`
     - 5
     - МНК-парабола по 5 точкам
     - :math:`a = \dfrac{2y_1 - y_2 - 2y_3 - y_4 + 2y_5}{14}`
     - Шум, Hanning kernel
   * - **5EA** ★★
     - :file:`discr5ea.h`
     - 5
     - МНК-Гаусс по 5 точкам
     - то же, но :math:`z_i = \ln y_i`
     - **Универсал, FFT**

★ — добавлено при рефакторинге 2026, ★★ — **рекомендуемый по умолчанию**
для FFT-задачи.

   Главный эксперимент: 5pt-методы стабильно лучше 3pt при любом SNR
