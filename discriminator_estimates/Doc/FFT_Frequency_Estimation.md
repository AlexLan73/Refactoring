## 1. Постановка задачи

### Что имеем

```
Входной сигнал: ~1.3 млн комплексных точек (I/Q данные)
Частота дискретизации: fs (Гц)
Задача: найти частоту максимального пика в спектре
         с точностью ЛУЧШЕ чем шаг по частоте (суб-бинная оценка)
```

### Текущий pipeline

```
Сигнал (1.3M точек)
    ↓
Zero-pad до N = 2^i (ближайшая степень двойки)
    ↓
[Опционально] Доп. zero-pad ×2 или ×4
    ↓
Окно Hanning
    ↓
FFT → N комплексных коэффициентов X(k)
    ↓
Магнитуда |X(k)| для k = 0..N/2
    ↓
Поиск максимума в окрестности ±1000 бинов
    ↓
❓ Уточнение частоты (суб-бинная оценка) ← ВОТ ЭТО ОБСУЖДАЕМ
```

### Зачем нужна суб-бинная оценка?

Без интерполяции точность = ±0.5 бина = ±Δf/2.
Для N = 4M, fs = 100 МГц: Δf = 25 Гц, ошибка ±12.5 Гц.

С интерполяцией: ошибка ±0.01·Δf = **±0.25 Гц** (улучшение в 50 раз!).

## 2. Базовые понятия: FFT, бины, шаг

### Что такое "бин" (bin)

**Бин** — одна ячейка (отсчёт) в FFT-спектре.

```
FFT из N точек даёт N комплексных коэффициентов: X(0), X(1), ..., X(N-1)

Каждый бин k соответствует частоте:
    f(k) = k · fs / N    (Гц)

Шаг по частоте (bin spacing):
    Δf = fs / N           (Гц)
```

**Пример**:
- fs = 100 МГц, N = 4 194 304 (= 2^22)
- Δf = 100 000 000 / 4 194 304 ≈ **23.84 Гц**
- Бин k = 1000 → частота 23 842 Гц

### Что такое "шаг"

В контексте нашей задачи **"шаг" = расстояние между соседними бинами**:

| Ситуация | Шаг (в бинах) | Шаг (в Гц) |
|----------|---------------|-------------|
| Без zero-padding | 1.0 | Δf |
| Zero-pad ×2 | 0.5 | Δf/2 |
| Zero-pad ×4 | 0.25 | Δf/4 |

### Почему шаг важен для интерполяции?

Если истинная частота f₀ попала МЕЖДУ двумя бинами, ближайший бин даёт
**грубую оценку** с ошибкой до:

```
Без zero-pad:  макс. ошибка = 0.5 бина = Δf/2
С zero-pad ×2: макс. ошибка = 0.25 бина = Δf/4
С zero-pad ×4: макс. ошибка = 0.125 бина = Δf/8
```

Чем меньше грубая ошибка → тем точнее работает интерполяция.

## 3. Zero-padding: что даёт и чего не даёт

### Что такое zero-padding

Дополнение сигнала нулями перед FFT:

```python
signal = [x₀, x₁, ..., x_{M-1}]           # M отсчётов
padded = [x₀, x₁, ..., x_{M-1}, 0, 0, ..., 0]  # N отсчётов, N > M
```

### Что ДАЁТ zero-padding

1. **Уплотняет спектральные отсчёты** — бины становятся чаще
2. **Интерполирует DTFT** — между "настоящими" бинами появляются промежуточные
3. **Улучшает интерполяцию пика** — ближайший бин ближе к истинной частоте
4. **N = 2^i** → можно использовать быстрый FFT (O(N log N) vs O(N²))

### Чего НЕ ДАЁТ zero-padding

**Не улучшает спектральное разрешение!**

Два близких пика на расстоянии < Δf_original **НЕ разделятся** при zero-padding.
Разрешение определяется длиной исходного сигнала M, а не длиной FFT N.

```
Спектральное разрешение ≈ fs / M (определяется длиной записи)
Шаг бинов = fs / N (определяется длиной FFT)

Zero-padding уменьшает шаг бинов, но НЕ разрешение!
```

### Визуальная аналогия

```
Без zero-pad:    •   •   •   •   •      ← редкие точки на кривой
С zero-pad ×2:   •·•·•·•·•·•·•·•·•      ← густые точки на ТОЙ ЖЕ кривой
С zero-pad ×4:   ••••••••••••••••••      ← ещё гуще, но кривая та же!
```

Кривая (DTFT) не меняется — меняется только плотность точек на ней.

## 4. Оконные функции и форма пика

### Зачем нужно окно?

Без окна (rectangular window) спектральный пик имеет форму **sinc(x) = sin(x)/x**:
- Узкий главный лепесток (2 бина)
- **Большие боковые лепестки** (-13 дБ = 20% от пика!)
- Боковые лепестки маскируют слабые сигналы рядом

### Окно Hanning (наш выбор)

```
w(n) = 0.5 · (1 - cos(2πn/N)),  n = 0, 1, ..., N-1
```

| Параметр | Rectangular | **Hanning** | Blackman |
|----------|------------|-------------|----------|
| Главный лепесток | 2 бина | **4 бина** | 6 бинов |
| Боковые лепестки | -13 дБ | **-31 дБ** | -58 дБ |
| Scalloping loss | 3.92 дБ | **1.42 дБ** | 1.10 дБ |
| Подходит для | Точность частоты | **Общее применение** | Динамический диапазон |

### Форма пика при Hanning

```
При rectangular: |X(f)| ≈ |sinc(π·δ)|         ← узкий, с "ушами"
При Hanning:     |X(f)| ≈ |sinc(π·δ)| · "сглаженный" ← широкий, без "ушей"
```

**Критическое свойство**: форма пика Hanning **гораздо ближе к параболе**,
чем sinc(x) при rectangular!

| Окно | Bias параболической интерполяции |
|------|----------------------------------|
| Rectangular | до ±0.05 бина (5%) |
| **Hanning** | **до ±0.005 бина (0.5%)** |
| Gaussian (дБ) | 0 (точно!) |
| Blackman | до ±0.001 бина (0.1%) |

**Вывод**: для Hanning параболическая интерполяция работает **очень хорошо!**

### Что Hanning делает с соседними пиками

Боковые лепестки при Hanning = -31 дБ (0.03 от пика).
Это значит: если есть слабый сигнал на уровне -30 дБ от основного —
Hanning его "спрячет". Для нашей задачи (один пик) это OK —
нам нужно подавить spectral leakage.

## 5. Методы интерполяции пика (по магнитуде)

> Эти методы работают с **|X(k)|** — магнитудой (амплитудой) спектра.
> Не требуют фазовой информации. Подходят для текущего pipeline.

### 5.1. Параболическая интерполяция по 3 точкам (Parabolic 3pt)

**Самый распространённый метод. Аналог нашего QA.**

Берём 3 бина вокруг максимума:

```
α = |X(k-1)|    ← левый сосед
β = |X(k)|      ← максимум
γ = |X(k+1)|    ← правый сосед
```

Формула вершины параболы:

```
         α - γ
δ = ─────────────────
    2 · (α - 2β + γ)

f_estimated = (k + δ) · fs / N
```

где δ ∈ [-0.5, +0.5] — смещение от бина k.

**Оценка амплитуды пика:**

```
A_peak = β - 0.25 · (α - γ) · δ
```

| Параметр | Значение |
|----------|----------|
| Точек | 3 |
| Bias (Hanning) | ±0.005 бина |
| Bias (Rectangular) | ±0.05 бина |
| Шумоустойчивость | Средняя |
| Сложность | O(1) — 4 операции |

**Когда использовать**: базовый метод, хорош для Hanning, прост в реализации.

### 5.2. Гауссова интерполяция по 3 точкам (Gaussian 3pt)

**Аналог нашего EA. Работает в логарифмическом масштабе.**

```
α = ln|X(k-1)|  (или 20·log₁₀|X(k-1)| в дБ)
β = ln|X(k)|
γ = ln|X(k+1)|

         α - γ
δ = ─────────────────
    2 · (α - 2β + γ)

f_estimated = (k + δ) · fs / N
```

**Формула одинаковая!** Разница только в том, что α, β, γ — логарифмы магнитуд.

**Почему это другой метод**: фитим не параболу `y = ax² + bx + c`,
а гауссиан `y = A·exp(-a(x-x₀)²)`, что через логарифм = парабола.

| Параметр | Значение |
|----------|----------|
| Bias (Hanning) | ±0.003 бина |
| Bias (Gaussian window + дБ) | **0 (точно!)** |
| Шумоустойчивость | Средняя |
| Ограничение | Все |X(k)| > 0 |

**Когда использовать**: когда форма пика ближе к гауссиану (Hanning, Blackman).

### 5.3. МНК-парабола по 5 точкам (LSQ Parabolic 5pt) — НОВЫЙ

**Метод наименьших квадратов. Основная тема нашего обсуждения.**

Берём 5 бинов вокруг максимума:

```
y₁ = |X(k-2)|, y₂ = |X(k-1)|, y₃ = |X(k)|, y₄ = |X(k+1)|, y₅ = |X(k+2)|
```

Фитим параболу `y = a·x² + b·x + c` методом наименьших квадратов.

**Ключевое отличие от 3-точечной**: парабола **НЕ обязана проходить** через все 5 точек.
Она минимизирует сумму квадратов отклонений → **сглаживает шум!**

**Замкнутая формула** (для равноотстоящих точек, xi = {-2, -1, 0, 1, 2} · step):

```
a = (2·y₁ - y₂ - 2·y₃ - y₄ + 2·y₅) / 14

b = (-2·y₁ - y₂ + y₄ + 2·y₅) / 10

      -b          шаг бинов
δ = ───── · ─────────────────
      2a     оригинальный шаг

f_estimated = (k + δ) · fs / N_padded
```

**Почему МНК-3 = обычная интерполяция**: 3 точки + 3 параметра (a, b, c) →
система определена точно → парабола проходит через все 3 точки → это НЕ МНК,
а точная интерполяция (= метод 5.1). **Нет выигрыша от "МНК" по 3 точкам!**

**Почему МНК-5 — это новое**: 5 точек + 3 параметра → переопределённая система →
настоящая аппроксимация → **сглаживание шума**.

| Параметр | Значение |
|----------|----------|
| Точек | 5 |
| Bias (Hanning + zp×2) | ±0.002 бина |
| Шумоустойчивость | **Высокая** |
| Сложность | O(1) — 10 операций |

**Когда использовать**: шумные данные, zero-padding ×2 (шаг = 0.5 бина).

### 5.4. МНК-Гауссиан по 5 точкам (LSQ Gaussian 5pt) — НОВЫЙ

Комбинация идей 5.2 и 5.3: **логарифмируем + МНК по 5 точкам**.

```
z₁ = ln|X(k-2)|, z₂ = ln|X(k-1)|, z₃ = ln|X(k)|,
z₄ = ln|X(k+1)|, z₅ = ln|X(k+2)|

a = (2·z₁ - z₂ - 2·z₃ - z₄ + 2·z₅) / 14
b = (-2·z₁ - z₂ + z₄ + 2·z₅) / 10
δ = -b / (2·a)      ← в единицах шага
```

| Параметр | Значение |
|----------|----------|
| Bias (Hanning) | ±0.001 бина |
| Шумоустойчивость | **Высокая** |
| Ограничение | Все |X(k)| > 0 |

**Лучший метод по магнитуде для Hanning!**

### 5.5. Взвешенный МНК по 5 точкам (Weighted LSQ 5pt)

Усовершенствование 5.3: даём точкам веса пропорционально амплитуде.

```
Веса: wi = yi²   (квадрат магнитуды)

Минимизируем: Σ wi · (yi - a·xi² - b·xi - c)²
```

Центральные точки (большая амплитуда) доминируют → парабола фитится по центру пика,
где она наиболее точна. Крайние точки (малая амплитуда, больше шума) почти не влияют.

**Замкнутой формулы нет** — нужно решать систему 3×3:

```
(Aᵀ·W·A) · [a, b, c]ᵀ = Aᵀ·W·y

где W = diag(w₁, w₂, w₃, w₄, w₅),  A = [xi², xi, 1]
```

Для фиксированных весов можно предвычислить обратную матрицу.

## 6. Методы по комплексным DFT-коэффициентам

> Эти методы используют **комплексные** X(k) — и амплитуду, и фазу.
> Они точнее магнитудных методов, потому что извлекают больше информации.
>
> **Для будущего проекта** (GPUWorkLib), где работаем с комплексными данными.

### 6.1. Estimator Якобсена (Jacobsen)

**Простейший из "умных" методов.**

```
X₋ = X(k-1)    ← комплексное значение
X₀ = X(k)      ← комплексное значение
X₊ = X(k+1)    ← комплексное значение

        ⎡ X₋ - X₊ ⎤
δ = Re ⎢───────────────⎥
        ⎣ 2·X₀ - X₋ - X₊ ⎦

f_estimated = (k + δ) · fs / N
```

Вся формула — одна комплексная дробь + взятие Re.

| Параметр | Значение |
|----------|----------|
| Bias | ±1% бина |
| Шум | Хорошая устойчивость |
| Сложность | O(1), 1 комплексное деление |
| Окно | Rectangular (для других нужна коррекция) |

### 6.2. Estimator Кандана (Candan) — bias-corrected Якобсен

**Исправляет систематическую ошибку Якобсена.**

```
        ⎡ X₋ - X₊ ⎤
δ_jac = Re ⎢───────────────⎥       ← Якобсен
        ⎣ 2·X₀ - X₋ - X₊ ⎦

     tan(π/N)
δ = ────────── · δ_jac              ← поправка Кандана
       π/N
```

Поправочный множитель `tan(π/N)/(π/N)`:
- Для N = 1024: ≈ 1.0000032
- Для N = 4M: ≈ 1 + 1.9×10⁻¹³ (практически 1)

**Для больших N поправка пренебрежимо мала!** Якобсен ≈ Кандан.

Но при малых N (< 256) поправка существенна.

| Параметр | Значение |
|----------|----------|
| Bias | **±0.1% бина** |
| Шум | Хорошая устойчивость |
| Сложность | O(1) |

### 6.3. Estimator Куинна (Quinn, 1st и 2nd)

**Наиболее точный 3-бинный метод. Достигает Cramér-Rao bound.**

**Quinn 1st** (упрощённый):

```
α₁ = Re[X(k+1) / X(k)]
α₂ = Re[X(k-1) / X(k)]

d₁ = α₁ / (1 - α₁)
d₂ = -α₂ / (1 - α₂)

Если d₁ > 0 и d₂ > 0:  δ = d₁
Если d₁ ≤ 0 и d₂ ≤ 0:  δ = d₂
Иначе:                   δ = (d₁ + d₂) / 2
```

**Quinn 2nd** (полный, с поправочной функцией τ):

```
τ(x) = 0.25 · ln(3x² + 6x + 1)
     - (√6/24) · ln( (x + 1 - √(2/3)) / (x + 1 + √(2/3)) )

δ = (d₁ + d₂)/2 + τ(d₁²) - τ(d₂²)
```

| Параметр | Значение |
|----------|----------|
| Bias | **±0.01% бина** |
| Шум | **Лучший** (достигает CRLB) |
| Сложность | O(1), ~20 операций |
| Окно | Rectangular (оптимален) |

### Что такое Cramér-Rao Lower Bound (CRLB)?

**CRLB** — теоретический предел точности оценки частоты при данном SNR.
Ни один метод не может быть точнее CRLB. Quinn 2nd достигает этого предела
при SNR > ~10 дБ.

```
Дисперсия оценки ≥ CRLB = 12 · fs² / (4π² · N · (N²-1) · SNR)

Для N = 4M, SNR = 20 дБ (= 100):
  CRLB ≈ 4.7×10⁻²³ · fs²  → σ_f ≈ 2.2×10⁻¹² · fs  (теоретический предел)
```

## 7. Итеративные и продвинутые методы

### 7.1. Итеративная параболическая интерполяция (IPI, 2025)

**Свежий метод** из статьи 2025 года. Итеративно уточняет оценку.

```
Алгоритм:
1. Грубая оценка: k_max = argmax|X(k)|
2. Шаг δ = 0.5 (половина бина)
3. Вычислить X(k_max - δ), X(k_max), X(k_max + δ)  ← через DTFT!
4. Параболическая интерполяция → новая оценка f_est
5. Уменьшить δ, сдвинуть центр в f_est
6. Повторить шаги 3-5 до сходимости (3-5 итераций)
```

**Ключевая идея**: на каждой итерации "зумируем" в пик, уменьшая интервал.
Требует вычисления DTFT в произвольных точках (не только в бинах FFT).

| Параметр | Значение |
|----------|----------|
| Точность | Достигает CRLB |
| Итераций | 3-5 |
| Сложность | O(N) на итерацию (DTFT) |

### 7.2. Chirp-Z Transform (CZT) / Zoom FFT

**"Увеличительное стекло" для спектра.**

Вместо FFT по всему диапазону [0, fs] — вычисляем спектр
только в узкой полосе [f₁, f₂] с произвольным разрешением.

```python
from scipy.signal import czt

# Zoom в полосу [f1, f2] с высоким разрешением
f1, f2 = peak_freq - 10*df, peak_freq + 10*df
W = np.exp(-2j * np.pi * (f2 - f1) / (M * fs))
A = np.exp(2j * np.pi * f1 / fs)
X_zoom = czt(signal, M=1024, W=W, A=A)
```

| Параметр | Значение |
|----------|----------|
| Точность | Произвольная (зависит от параметров) |
| Сложность | O(N log N) |
| Гибкость | Максимальная |

### 7.3. DTFT в произвольной точке

Простейший "продвинутый" метод — вычислить DTFT напрямую:

```
           N-1
X(f) = Σ  x(n) · exp(-j·2π·f·n/fs)
          n=0
```

Это O(N) операций для одной частоты.
Для уточнения пика: вычислить X(f) в 3-5 точках вокруг грубой оценки.

**Простой и мощный метод!** Для N = 4M и 5 точек: 20M умножений.

## 8. Суперразрешение (несколько сигналов)

> Если нужно найти **несколько близких частот** в шуме.
> Не нужно для текущей задачи (один пик), но полезно знать.

### 8.1. MUSIC (Multiple Signal Classification)

- Строит корреляционную матрицу сигнала
- Раскладывает на "сигнальное" и "шумовое" подпространства
- Разрешение лучше чем Δf (суперразрешение!)
- Нужно знать количество сигналов

### 8.2. ESPRIT (Estimation of Signal Parameters via Rotational Invariance)

- Похож на MUSIC, но автоматически определяет параметры
- Не требует "сканирования" по частоте
- Более устойчив к шуму

### 8.3. Когда использовать

- MUSIC/ESPRIT: несколько синусоид с разностью частот < Δf
- FFT + интерполяция: один сигнал или далеко разнесённые сигналы

## 10. Рекомендации для нашей задачи

### Текущий pipeline (CPU, магнитуда, Hanning)

**Рекомендация: LSQ Gaussian 5pt (метод 4) + zero-padding ×2**

Почему:
1. Hanning делает форму пика "параболоидной" → bias минимальный
2. МНК по 5 точкам сглаживает шум (у нас его "очень много")
3. Логарифмическая шкала (Gaussian) лучше моделирует форму пика Hanning
4. Замкнутая формула — O(1), мгновенно на CPU
5. Zero-pad ×2 обеспечивает все 5 точек в главном лепестке

**Fallback**: если |X(k)| ≈ 0 у крайних точек (логарифм не определён) →
переключиться на LSQ Parabolic 5pt (метод 3).

### Будущий pipeline (GPU, комплексные данные)

**Рекомендация: Quinn 2nd estimator (метод 8)**

Почему:
1. Использует фазу → больше информации → лучше при шуме
2. Достигает Cramér-Rao bound (теоретический предел)
3. Всего 3 бина — минимальная передача данных GPU → CPU
4. Элементарная формула — легко параллелизуется

### Резервный вариант для особо шумных данных

**CZT (Zoom FFT)** вокруг грубой оценки пика:
1. FFT → грубая оценка k_max
2. CZT в окрестности k_max ± 5 бинов с разрешением ×100
3. Поиск максимума в "зумированном" спектре

## 11. Примеры кода (Python)

### 11.1. Parabolic 3pt (текущий QA)

```python
def parabolic_3pt(X_mag, k):
    """
    Параболическая интерполяция по 3 точкам.

    X_mag: массив |X(k)| — магнитуда FFT
    k: индекс максимального бина
    return: (delta, amplitude) — смещение от бина k и амплитуда пика
    """
    alpha = X_mag[k - 1]
    beta  = X_mag[k]
    gamma = X_mag[k + 1]

    denom = alpha - 2.0 * beta + gamma
    if abs(denom) < 1e-30:
        return 0.0, beta

    delta = 0.5 * (alpha - gamma) / denom
    amp   = beta - 0.25 * (alpha - gamma) * delta
    return delta, amp
```

### 11.2. Gaussian 3pt (текущий EA)

```python
import numpy as np

def gaussian_3pt(X_mag, k):
    """
    Гауссова интерполяция по 3 точкам (в лог-масштабе).

    X_mag: массив |X(k)| — магнитуда FFT
    k: индекс максимального бина
    return: delta — смещение от бина k
    """
    if X_mag[k-1] <= 0 or X_mag[k] <= 0 or X_mag[k+1] <= 0:
        return parabolic_3pt(X_mag, k)[0]  # fallback

    alpha = np.log(X_mag[k - 1])
    beta  = np.log(X_mag[k])
    gamma = np.log(X_mag[k + 1])

    denom = alpha - 2.0 * beta + gamma
    if abs(denom) < 1e-30:
        return 0.0

    return 0.5 * (alpha - gamma) / denom
```

### 11.3. МНК-парабола по 5 точкам (НОВЫЙ)

```python
def lsq_parabolic_5pt(X_mag, k):
    """
    МНК-параболическая интерполяция по 5 точкам.

    X_mag:    массив |X(k)| — магнитуда FFT
    k:        индекс максимального бина (в padded-спектре)
    return:   delta — смещение от бина k (в padded-индексах)

    Формула для равноотстоящих точек xi = {-2, -1, 0, 1, 2}·step:
        a = (2·y1 - y2 - 2·y3 - y4 + 2·y5) / 14
        b = (-2·y1 - y2 + y4 + 2·y5) / 10
        delta = -b / (2·a)
    """
    y1 = X_mag[k - 2]
    y2 = X_mag[k - 1]
    y3 = X_mag[k]
    y4 = X_mag[k + 1]
    y5 = X_mag[k + 2]

    a = (2.0*y1 - y2 - 2.0*y3 - y4 + 2.0*y5) / 14.0
    b = (-2.0*y1 - y2 + y4 + 2.0*y5) / 10.0

    if abs(a) < 1e-30:
        return 0.0

    delta = -b / (2.0 * a)  # в единицах шага между точками (= 1 padded bin)
    return delta              # смещение в padded-индексах (для f = (k+δ)·fs/N)
```

### 11.4. МНК-Гауссиан по 5 точкам (НОВЫЙ — РЕКОМЕНДУЕМЫЙ)

```python
def lsq_gaussian_5pt(X_mag, k):
    """
    МНК-Гауссова интерполяция по 5 точкам (в лог-масштабе).

    Лучший метод для Hanning window + шумные данные.

    X_mag:    массив |X(k)| — магнитуда FFT
    k:        индекс максимального бина (в padded-спектре)
    return:   delta — смещение от бина k (в padded-индексах)
    """
    # Проверка: все амплитуды > 0 (нужно для логарифма)
    vals = [X_mag[k+i] for i in range(-2, 3)]
    if any(v <= 0 for v in vals):
        return lsq_parabolic_5pt(X_mag, k)  # fallback

    z1 = np.log(X_mag[k - 2])
    z2 = np.log(X_mag[k - 1])
    z3 = np.log(X_mag[k])
    z4 = np.log(X_mag[k + 1])
    z5 = np.log(X_mag[k + 2])

    a = (2.0*z1 - z2 - 2.0*z3 - z4 + 2.0*z5) / 14.0
    b = (-2.0*z1 - z2 + z4 + 2.0*z5) / 10.0

    if abs(a) < 1e-30:
        return 0.0

    delta = -b / (2.0 * a)
    return delta  # смещение в padded-индексах (для f = (k+δ)·fs/N)
```

### 11.5. Полный pipeline

```python
import numpy as np

def estimate_peak_frequency(signal, fs, zero_pad_factor=2, search_radius=1000):
    """
    Полный pipeline определения частоты пика в спектре.

    signal:          комплексный сигнал (1D array)
    fs:              частота дискретизации (Гц)
    zero_pad_factor: множитель zero-padding (1, 2, 4)
    search_radius:   радиус поиска пика (в бинах)
    return:          частота пика (Гц)
    """
    M = len(signal)

    # 1. Zero-pad до ближайшей степени 2
    N_base = 1
    while N_base < M:
        N_base <<= 1

    # 2. Дополнительный zero-pad
    N = N_base * zero_pad_factor

    # 3. Окно Hanning (np.hanning deprecated в NumPy 2.0+, alt: scipy.signal.windows.hann)
    window = np.hanning(M)
    windowed = signal * window

    # 4. FFT с zero-padding
    X = np.fft.fft(windowed, n=N)
    X_mag = np.abs(X[:N // 2])  # только положительные частоты

    # 5. Грубый поиск максимума (в заданном диапазоне)
    k_start = max(2, search_radius if search_radius else 2)
    k_end   = min(len(X_mag) - 3, k_start + 2 * search_radius if search_radius else len(X_mag) - 3)
    k_max   = k_start + np.argmax(X_mag[k_start:k_end])

    # 6. Суб-бинная оценка (МНК-Гауссиан по 5 точкам)
    if k_max >= 2 and k_max < len(X_mag) - 2:
        delta = lsq_gaussian_5pt(X_mag, k_max)
    else:
        delta = parabolic_3pt(X_mag, k_max)[0]  # fallback для краёв

    # 7. Частота
    f_estimated = (k_max + delta) * fs / N
    return f_estimated


# --- Пример использования ---
if __name__ == "__main__":
    fs = 100e6         # 100 МГц
    f_true = 1.23456e6 # 1.23456 МГц — истинная частота
    duration = 0.013   # 1.3M точек при 100 МГц
    N_samples = int(fs * duration)

    t = np.arange(N_samples) / fs
    noise = 0.5 * (np.random.randn(N_samples) + 1j * np.random.randn(N_samples))
    signal = np.exp(2j * np.pi * f_true * t) + noise

    f_est = estimate_peak_frequency(signal, fs, zero_pad_factor=2)

    print(f"Истинная частота: {f_true:.2f} Гц")
    print(f"Оценка:          {f_est:.2f} Гц")
    print(f"Ошибка:          {abs(f_est - f_true):.4f} Гц")
    print(f"Ошибка:          {abs(f_est - f_true) / (fs / N_samples):.6f} бин")
```

### 11.6. Якобсен (для будущего комплексного варианта)

```python
def jacobsen_estimator(X_complex, k, N, fs):
    """
    Estimator Якобсена — по комплексным DFT коэффициентам.

    X_complex: комплексный FFT-спектр
    k:         индекс максимального бина
    N:         длина FFT
    fs:        частота дискретизации
    return:    частота пика (Гц)
    """
    Xm = X_complex[k - 1]  # X(k-1)
    X0 = X_complex[k]      # X(k)
    Xp = X_complex[k + 1]  # X(k+1)

    denom = 2.0 * X0 - Xm - Xp
    if abs(denom) < 1e-30:
        return k * fs / N

    delta = (Xm - Xp) / denom
    delta = delta.real  # берём вещественную часть

    return (k + delta) * fs / N
```

### 11.7. Quinn 2nd (для будущего — максимальная точность)

```python
def quinn2_estimator(X_complex, k, N, fs):
    """
    Quinn's 2nd Estimator — достигает Cramér-Rao bound.

    X_complex: комплексный FFT-спектр
    k:         индекс максимального бина
    N:         длина FFT
    fs:        частота дискретизации
    return:    частота пика (Гц)
    """
    def tau(x):
        """Поправочная функция Quinn."""
        return (0.25 * np.log(3.0*x*x + 6.0*x + 1.0)
                - np.sqrt(6.0)/24.0
                * np.log((x + 1.0 - np.sqrt(2.0/3.0))
                       / (x + 1.0 + np.sqrt(2.0/3.0))))

    X0 = X_complex[k]
    Xp = X_complex[k + 1]
    Xm = X_complex[k - 1]

    ap = (Xp / X0).real
    am = (Xm / X0).real

    d1 = ap / (1.0 - ap)
    d2 = -am / (1.0 - am)

    delta = (d1 + d2) / 2.0 + tau(d1 * d1) - tau(d2 * d2)

    return (k + delta) * fs / N
```

## 12. Связь с модулем discriminator_estimates

### Общая математика

Модуль discriminator_estimates решает **ту же задачу** — нахождение максимума функции
по дискретным отсчётам. Разница только в контексте:

| Параметр | discriminator_estimates | FFT frequency estimation |
|----------|------------------------|--------------------------|
| Что ищем | Координату ДН антенны | Частоту пика в спектре |
| Форма сигнала | sinc(x) = sin(x)/x | Зависит от оконной функции |
| Отсчёты | Амплитуды ДН | Магнитуда |X(k)| |
| "Шаг" | Шаг сканирования | Δf = fs/N |

### Соответствие методов

| discriminator_estimates | FFT-аналог |
|------------------------|------------|
| `discr3cg` (центр тяжести) | Centroid (взвешенное среднее бинов) |
| `discr2sd` (суммарно-разностный) | — |
| `discr3qa` (квадратичная аппр.) | Parabolic 3pt interpolation |
| `discr3ea` (экспоненциальная аппр.) | Gaussian 3pt interpolation |
| **discr5qa** (МНК-парабола 5pt) | **LSQ Parabolic 5pt** ← TODO: не реализован |
| **discr5ea** (МНК-Гауссиан 5pt) | **LSQ Gaussian 5pt** ← реализован ✅ |

### Переиспользование кода

Функции `discr5qa` и `discr5ea` можно использовать **и для ДН, и для FFT**.
Они не зависят от контекста — это чистая математика нахождения вершины параболы
по 5 дискретным точкам.
