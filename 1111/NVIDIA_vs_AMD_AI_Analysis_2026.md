## 3. Почему NVIDIA быстрее: два фактора

**Фактор 1: Bandwidth.**  
RTX 5060 Ti: 448 GB/s (GDDR7). RX 9070 XT: ~576 GB/s (GDDR6). Казалось бы, AMD впереди. Но:

**Фактор 2: Софтовая оптимизация (главный).**  
CUDA + llama.cpp/Ollama оптимизированы 10+ лет. Каждая операция (матричное умножение, KV-cache, attention) имеет специализированные CUDA-ядра. ROCm догоняет, но на практике AI-фреймворки на AMD дают на 30–50% меньше производительности при сравнимом железе:

- **Tom's Hardware:** RTX 5060 Ti 16GB даёт 40% прирост tok/s по сравнению с RTX 4060 Ti 16GB
- **dropreference.com:** «AI is a priority — stick with Nvidia in 2026. Many AI frameworks do not natively support AMD, performance is often 30–50% lower»
- **Hardware Corner:** RTX 5080 на LLM-бенчмарках — 100% baseline; RTX 4090 — 123% (нормализованный)

> **Для чайников:** NVIDIA — как автобан с идеальным покрытием. AMD — как просёлочная дорога: доедете, но медленнее и трясёт. Мотор (железо) у обоих мощный, но дорога (софт) решает.

## 4. Linux vs Windows для AI

### 4.1. Сводная таблица

| Аспект | Linux (Ubuntu 24.04+) | Windows 11 |
|--------|:-:|:-:|
| **NVIDIA (CUDA)** | Из коробки. Ollama, llama.cpp, PyTorch | Из коробки. Ollama, llama.cpp, PyTorch |
| **AMD (ROCm 7.2+)** | Из коробки для RX 9070 XT (gfx1201) | Ручная установка HIP SDK 7.1+, подмена библиотек rocblas |
| **Fine-tuning NVIDIA** | Unsloth, Axolotl, LLaMA-Factory — работают | Работает, но некоторые пакеты сложнее ставить |
| **Fine-tuning AMD** | Ограниченно (PyTorch ROCm build) | Практически невозможно |
| **Docker** | Нативная поддержка. GPU passthrough работает | WSL2 + Docker Desktop. GPU passthrough сложнее |
| **vLLM (серверный inference)** | Полная поддержка | Ограниченная |
| **MCP-серверы** | Работают нативно | Работают через Node.js |
| **Cline + Ollama** | Идеально | Идеально (NVIDIA). Проблемы (AMD) |
| **Потребление RAM на ОС** | ~1–2 GB | ~4–6 GB |
| **Стабильность 24/7** | Серверный стандарт | Обновления Windows могут прервать |

### 4.2. Что выбрать

| Ваш GPU | Рекомендация ОС | Почему |
|---------|:-:|---------|
| NVIDIA (любая) | **Без разницы** | CUDA одинаково хорош на Linux и Windows |
| AMD RX 9070 XT | **Только Linux** | ROCm на Windows — ручная настройка без гарантий |
| Сервер (24/7, multi-user) | **Linux** | Стабильность, Docker, vLLM, меньше расход RAM |
| Рабочая станция (1 чел.) | **Что привычнее** | Если NVIDIA — ОС не критична |

### 4.3. Проблемы AMD на Windows — конкретика

На январь–февраль 2026 на GitHub зафиксированы массовые issues:
- **ollama/ollama#13920:** Ollama детектирует RX 9070 XT, но не инициализирует — «filtering device which didn't fully initialize». Падает на CPU-inference.
- **ollama/ollama#11542:** GPU idle 0%, CPU на 100%. Ollama молча откатывается на CPU без предупреждения.
- **Решение (doroch.com):** Ручная установка HIP SDK 7.1, подмена rocblas библиотек из `C:\Program Files\AMD\ROCm\7.1\bin\rocblas\library` в папку Ollama. Работает, но процедура не тривиальная.
- **Linux:** На Ubuntu 24.04 + ROCm 6.4.1+ «Ollama docker just works» (подтверждено khairil yusof, практические тесты).

> **Для чайников:** на NVIDIA (Windows или Linux) вы устанавливаете Ollama и через 2 минуты работаете. На AMD + Windows — от 2 часов до «не получилось». На AMD + Linux — 10 минут.

## 5. Где AMD всё-таки выигрывает (честный анализ)

1. **Игры.** RX 9070 XT обгоняет RTX 5070 ($649) в играх на 41% (3DMark). Если GPU для игр И AI — AMD выгоднее для игр.
2. **Open-source стек.** ROCm полностью open-source (vs проприетарный CUDA).
3. **Перспектива.** Каждая версия ROCm лучше предыдущей. ROCm 7.2+ значительно ближе к CUDA чем ROCm 5.x.
4. **Если уже есть RX 9070 XT** — не нужно покупать NVIDIA специально для AI. Ollama на Linux работает, модели до 14B запускаются.

## 9. Глоссарий

| Термин | Объяснение | Для чайников |
|--------|-----------|-------------|
| **tok/s** | Токенов в секунду — скорость генерации AI | Скорость печати AI. 30 = комфортно. 60+ = мгновенно |
| **VRAM** | Память видеокарты | Чем больше — тем крупнее AI-модель влезет |
| **Bandwidth** | Пропускная способность памяти (GB/s) | «Ширина трубы» — чем шире, тем быстрее данные текут |
| **CUDA** | Платформа NVIDIA для GPU-вычислений | «Язык», на котором программы говорят с видеокартой NVIDIA |
| **ROCm** | Аналог CUDA от AMD. Open-source | Тот же «язык», но для AMD. Моложе, менее отлажен |
| **NVFP4** | Новый формат квантизации Blackwell | Эксклюзив RTX 50xx. Даёт +60% скорости на AI |
| **Q4 / Q8** | Квантизация: 4-bit / 8-bit на параметр | Как JPEG: сильнее сжал = меньше файл, чуть хуже качество |
| **Офлоад** | Часть модели в VRAM, часть в RAM | Как книга на двух столах — приходится бегать туда-сюда |
| **Ollama** | Программа для запуска AI локально | VLC для AI. Скачал, одна команда, работает |
| **Fine-tuning** | Дообучение модели на ваших данных | Натаскивание собаки на ваши команды |
| **vLLM** | Серверный движок для AI inference | Ollama для серверов — много пользователей одновременно |
| **HIP SDK** | AMD-библиотеки для GPU на Windows | «Переводчик» между Windows и видеокартой AMD |
| **gfx1201** | Кодовое имя чипа RX 9070 XT | Как серийный номер — нужен чтобы ROCm знал какая карта |
